# Agent Shared Fabric v0.3.0

This release polishes the product model: Agent Shared Fabric now clearly separates the fixed governance core from customizable extension tools.

## Added

- executable hook templates:
  - `templates/governance-core/hooks/before-task.sh`
  - `templates/governance-core/hooks/log-phase.sh`
  - `templates/governance-core/hooks/after-task.sh`
- generated hook installation in `scripts/init_agent_shared_fabric.py`
- department routing documentation
- customization guide for MCP, skills, workflows, Maestro, and MemPalace

## Changed

- corrected the architecture flow so postflight writes memory lanes before receipts are consumed by Fabric App
- clarified that memory lanes also flow back into the next agent runtime
- redrew brain/body separation as internal fabric vs external tools around the runtime
- attributed the six-stage discipline as inspired by `cft0808/edict`
- reframed MemPalace and Maestro as strongly recommended integrations, not hard dependencies

## Verified

- smoke-tested generated hooks in a temporary local install
- verified `[BOOT_OK]` and `[SYNC_OK]` hook flow
- scanned public repository content for personal paths and obvious secret patterns
